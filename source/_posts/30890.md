<p>Salt Master外部Job Cache配置</p><p>Posted on 2014-11-24 in kb</p><p><br/></p><p>原文出处: <a href="http://pengyao.org/saltstack-master-external-job-cache.html" _src="http://pengyao.org/saltstack-master-external-job-cache.html">http://pengyao.org/saltstack-master-external-job-cache.html</a>&nbsp;|&nbsp;<a href="http://www.saltstack.cn/kb/saltstack_master_retuner_over_event_system/#saltstack_master_retuner_over_event_system" _src="http://www.saltstack.cn/kb/saltstack_master_retuner_over_event_system/#saltstack_master_retuner_over_event_system">http://www.saltstack.cn/kb/saltstack_master_retuner_over_event_system/#saltstack_master_retuner_over_event_system</a>&nbsp;</p><p>作者: pengyao</p><p>SaltStack 2014.7之前, Minion端的执行结果想存储在外部系统中, 需要借助returner进行配置. 而returner的工作方式是由minion端直接连接对应的returner, 在分布式环境中由于网络等限制,该方式并不友好.</p><p><br/></p><p>而在Master端, Job Cache会以文件的形式存储在Master本地磁盘, 对第三方系统并不友好. 基于此, 之前有分享过 基于Salt Event系统构建Master端returner , 需要启动另外一个进程, 进行监听Salt Event接口, 并将结果存储在第三方系统中. 刚好看到2014.7.0中master端增加了 master_job_cache参数, 可以直接外放Job Cache, 就做个测试, 测试下这个功能.</p><p><br/></p><p>环境说明</p><p><br/></p><p>Salt Version: 2014.7.0</p><p>OS: CentOS 6.5 X86_64 (with EPEL)</p><p>本次测试结果将存储在MySQL中, 为了方便测试, 已在Master本地部署了MySQL Server</p><p>开工</p><p><br/></p><p>前置配置</p><p><br/></p><p>安装MySQLdb依赖:</p><p><br/></p><p>yum -y install MySQL-python</p><p>配置本次测试需要使用的数据库及用户:</p><p><br/></p><p># 创建salt数据库</p><p>mysql -e &#39;create database salt&#39;</p><p># 创建用于连接salt数据库的用户</p><p>mysql -e &#39;&quot;grant all on salt.* to salt@localhost identified by &quot;salt_pass&#39;;</p><p># 将数据库配置添加至master配置文件中</p><pre class="brush:python;toolbar:false">mysql.host:&nbsp;&#39;localhost&#39;
mysql.user:&nbsp;&#39;root&#39;
mysql.pass:&nbsp;&#39;pass&#39;
mysql.db:&nbsp;&#39;salt&#39;
mysql.port:&nbsp;3306
master_job_cache:&nbsp;mysql&nbsp;//省去配置minion</pre><p><br/></p><p>创建用于存储Job的数据库表结构:</p><p><br/></p><p>USE `salt`;</p><p><br/></p><p>--</p><p>-- Table structure for table `jids`</p><p>--</p><p><br/></p><p>DROP TABLE IF EXISTS `jids`;</p><p>CREATE TABLE `jids` (</p><p>&nbsp; `jid` varchar(255) NOT NULL,</p><p>&nbsp; `load` mediumtext NOT NULL,</p><p>&nbsp; UNIQUE KEY `jid` (`jid`)</p><p>) ENGINE=InnoDB DEFAULT CHARSET=utf8;</p><p><br/></p><p>--</p><p>-- Table structure for table `salt_returns`</p><p>--</p><p><br/></p><p>DROP TABLE IF EXISTS `salt_returns`;</p><p>CREATE TABLE `salt_returns` (</p><p>&nbsp; `fun` varchar(50) NOT NULL,</p><p>&nbsp; `jid` varchar(255) NOT NULL,</p><p>&nbsp; `return` mediumtext NOT NULL,</p><p>&nbsp; `id` varchar(255) NOT NULL,</p><p>&nbsp; `success` varchar(10) NOT NULL,</p><p>&nbsp; `full_ret` mediumtext NOT NULL,</p><p>&nbsp; `alter_time` TIMESTAMP DEFAULT CURRENT_TIMESTAMP,</p><p>&nbsp; KEY `id` (`id`),</p><p>&nbsp; KEY `jid` (`jid`),</p><p>&nbsp; KEY `fun` (`fun`)</p><p>) ENGINE=InnoDB DEFAULT CHARSET=utf8;</p><p>配置Master</p><p><br/></p><p>将MySQL连接权限等信息添加到Salt Master配置文件中:</p><p><br/></p><p>echo -e &quot;\n\n# MySQL\nmysql.host: &#39;localhost&#39;\nmysql.user: &#39;salt&#39;\nmysql.pass: &#39;salt_pass&#39;\nmysql.db: &#39;salt&#39;\nmysql.port: 3306&quot; &gt;&gt; /etc/salt/master</p><p>配置master_job_cache选项, 以使将Job结果存储在MySQL中:</p><p><br/></p><p>echo -e &quot;\n\n# Master Job Cache\nmaster_job_cache: mysql&quot; &gt;&gt; /etc/salt/master</p><p>重启Salt Master, 以使配置生效:</p><p><br/></p><p>service salt-master restart</p><p>测试</p><p><br/></p><p>对主机执行test.ping:</p><p><br/></p><p>salt &#39;*&#39; test.ping -v</p><p>输出结果:</p><p><br/></p><p>Executing job with jid 20141120060202308159</p><p>-------------------------------------------</p><p><br/></p><p>minion-01.example.com:</p><p>&nbsp; &nbsp; True</p><p>查询MySQL jids表数据:</p><p><br/></p><p>mysql salt -e &#39;select * from jids\G&#39;</p><p>输出结果:</p><p><br/></p><p>*************************** 1. row ***************************</p><p>jid: 20141120060202308159</p><p>load: {&quot;tgt_type&quot;: &quot;glob&quot;, &quot;jid&quot;: &quot;20141120060202308159&quot;, &quot;cmd&quot;: &quot;publish&quot;, &quot;tgt&quot;: &quot;*&quot;, &quot;kwargs&quot;: {&quot;show_timeout&quot;: false, &quot;show_jid&quot;: false}, &quot;ret&quot;: &quot;&quot;, &quot;user&quot;: &quot;sudo_vagrant&quot;, &quot;arg&quot;: [], &quot;fun&quot;: &quot;test.ping&quot;}</p><p>查询MySQL salt_returns表数据:</p><p><br/></p><p>mysql salt -e &#39;select * from salt_returns\G&#39;</p><p>输出结果:</p><p><br/></p><p>*************************** 1. row ***************************</p><p>fun: test.ping</p><p>jid: 20141120060202308159</p><p>return: true</p><p>id: minion-01.example.com</p><p>success: 1</p><p>full_ret: {&quot;fun_args&quot;: [], &quot;jid&quot;: &quot;20141120060202308159&quot;, &quot;return&quot;: true, &quot;retcode&quot;: 0, &quot;success&quot;: true, &quot;cmd&quot;: &quot;_return&quot;, &quot;_stamp&quot;: &quot;2014-11-20T06:02:02.533850&quot;, &quot;fun&quot;: &quot;test.ping&quot;, &quot;id&quot;: &quot;minion-01.example.com&quot;}</p><p>alter_time: 2014-11-20 06:02:02</p><p>Job执行结果已经按照之前的配置存储到了MySQL中, 达到预期效果</p><p><br/></p>
